# AI 助手与老年顾问差异化方案

> FIT "国脉杯"软件设计大赛 · 队伍：冲冲冲
> 问题：AI 智能助手和老年理财顾问功能、UI 过于相似，评委会质疑差异化

---

## 一、问题诊断

当前两个页面的实质差异仅在于：
- `mode: 'elder'` 参数 → 后端 system prompt 追加 10 行"老年模式"指令
- 字号：1.0625rem → 1.25rem
- 默认开启自动朗读
- 快捷按钮文案不同

**本质问题**：两个页面都是"聊天机器人"，共用同一个 WebSocket、同一套工具集、同一种 UI 范式（消息气泡流）。

---

## 二、核心差异化策略

> **AI 助手 = "油门"（交易执行引擎），老年顾问 = "刹车"（风险守护引擎）**
>
> "不是帮老人管钱的工具，而是让子女放心、让老人自主的'家庭金融信任层'——AI 当参谋，子女当后盾，老人当主人。"

| 维度 | AI 智能助手 | 老年理财顾问 |
|------|-----------|------------|
| **定位** | 专业交易执行引擎 | 家庭风险守护顾问 |
| **目标用户** | 有投资经验的中青年用户 | 60+ 老年用户及其子女 |
| **核心能力** | 行情分析、策略回测、自动交易 | 资产概览、风险预警、只读建议 |
| **交互范式** | 聊天流 + 专业面板 | 语音卡片流 / 引导式问答 |
| **工具权限** | 完整（买入、卖出、定投、转账） | 只读（查余额、查行情、查动态） |
| **LLM 配置** | 快速模式、temperature=0.7 | 慢思考模式、temperature=0.3 |
| **联动** | 独立使用 | 联动监护中心（超限操作 → 子女确认） |

---

## 三、UI/UX 差异化方案

### 方案一（推荐）：仪表盘卡片式 vs 聊天式

**老年顾问**：放弃聊天流布局。页面主体为大卡片网格：
- 总资产卡片（超大数字 + 趋势箭头）
- 风险评级卡片（盾牌图标 + 等级文字）
- 近期提醒卡片（"您已连续30天未超额消费"）
- "问顾问"卡片（点击进入语音/文字交互）

点击任意卡片展开详情或引导式对话。

**AI 助手**：保持现行聊天流布局。增加侧边栏专业面板（技术指标、持仓概览）。

**区分效果**：布局范式完全不同（卡片阵列 vs 气泡列表），一眼可辨。

### 方案二：语音通话式 vs 文字聊天式

**老年顾问**：模拟"视频通话"界面。中央大圆形顾问头像 + 呼吸光环动画，底部巨大麦克风按钮（80px+），运行中支持语音打断（WebSocket interrupt），文字作为字幕显示。

**AI 助手**：保持文字输入为主的聊天框。

### 方案三：暖色调拟物 vs 冷色调科技

**老年顾问**：木纹/米色暖调背景，模拟"家庭顾问桌面"。回复以"便签条"样式呈现，右上角有"已确认"印章动画。顶部固定显示顾问姓名 + "从业20年·稳健型"资质标签。

**AI 助手**：保留冷灰白科技风格。

### 方案四（推荐）：引导问答式 vs 自由对话式

**老年顾问**：首屏展示 4 个超大 tappable 选项卡片：
- "我的钱有多少"（Wallet 图标）
- "哪些产品稳当"（ShieldCheck 图标）
- "家人最近在干嘛"（User 图标）
- "帮我分析风险"（AlertCircle 图标）

每步最多 3 个选项，像 ATM 机一样逐步引导。文字输入框收起为次级入口。

**AI 助手**：保留自由文本输入 + 快捷提示词标签。

### 方案五：报告单样式 vs 聊天记录式

**老年顾问**：每次回答生成格式化"财务咨询报告单"——标题栏（日期+报告编号）、正文（资产概况、风险提示、建议操作）、底部"顾问签名"。历史记录是一叠报告单缩略图。

**AI 助手**：连续气泡聊天流。

## 四、技术架构差异化方案

### 方案一（P0）：独立工具集 —— 只读 vs 读写

```
新增 ELDER_TOOLS（仅保留查询类）：
  - get_user_balance
  - get_market_analysis
  - get_user_activity

LLM_TOOLS 保持完整 + 新增专业工具：
  - buy_stock, set_dca_task（写操作，老年顾问不可用）
  - run_backtest（AI 助手专属）
```

后端根据 mode 参数选择不同工具集，不需要新增端点。

### 方案二（P0）：独立 LLM 配置

```
老年顾问：
  temperature: 0.3
  enable_thinking: true（慢思考）
  max_tokens: 1024

AI 助手：
  temperature: 0.7
  enable_thinking: false（快速模式）
```

### 方案三（P1）：elder_profiles 数据模型

```json
{
  "user_id": "xxx",
  "risk_profile": "conservative",
  "disabled_products": ["GMFT"],
  "daily_checkin_time": "09:00",
  "voice_preference": "slow"
}
```

### 方案四（P2）：主动语音推送

利用现有 tts.py，定时任务检查市场预警，WebSocket 推送至 ElderAdvisor 自动朗读。

### 方案五（P3）：AI 助手专属策略回测

新增 `run_backtest` 工具，前端用 Recharts 绘制收益曲线。

## 五、评委预判问答

1. **"怎么证明不是换皮？"** → 独立工具集（只读/读写）、独立 LLM 配置、不同的 UI 布局范式、监护中心联动拦截。本质是两种产品。

2. **"老人真会用聊天框？"** → 引导式大按钮 + 语音输入为主，文字输入为次级入口。借鉴 ATM 逐步引导模式。

3. **"和监护中心有什么联动？"** → 老年顾问感知监护额度，超额操作自动生成语音解释并推送子女确认，形成"AI 参谋 → 子女复核 → 老人决策"信任链。

## 六、实施优先级

| 优先级 | 方案 | 工时 | 效果 |
|--------|------|------|------|
| P0 | 独立工具集 | 2h | 安全隔离 |
| P0 | 独立 LLM 配置 | 1h | 回复风格差异 |
| P0 | 引导式卡片布局 | 4h | 视觉一眼区分 |
| P1 | elder_profiles 数据模型 | 3h | 个性化体验 |
| P1 | 语音交互强化 | 3h | 适老化亮点 |
| P2 | 主动语音推送 | 4h | 被动→主动 |
| P2 | 报告单样式 | 3h | 专业感 |
| P3 | 策略回测 | 6h | AI 助手专业化 |

**赛前建议完成 P0 全部 + P1 方案三，约 10 小时。**
